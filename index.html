Hi there!

Best UX UI (mobile)

I'm trying to get fixed navbar with two off-canvas menus, which will push the main content.
One on the left side and second on the right side.
It will be nice, if we could implement just one (left) menu as first.
Here is few requirements:

  1. Fixed navbar OK
  2. Fixed navbar on small viewports change to static (non-fixed)OK
  3. Offset menu can be larger than viewport height (scrollable) OK
  4. When hidding the menu, do not change menu scroll position NO
  5. When menu is triggered (shown):
    • always show at the top position (of menu) depending on 4.
    • user can't scroll the main content OK
    • fixed navbar shows properly on every scroll position on page NO
    • don't scroll content on the top NO
  6. Smooth scroll on iOS - content OK
  7. Smooth scroll on iOS - menu without using -webkit-overflow-scrolling: touch; on .st-menu-open .st-menu NO

Here is my reference example. Try the [PUSH] version/button. This is what I want to achieve. But in referenced example is not mitigating points:

  • fixed navbar
  • scrollable menu
  • smoothscroll on iOS

You may see this menu from (web)application like Facebook, Yahoo!, Google+ and so on, but all of them have slightly different behaviour.
